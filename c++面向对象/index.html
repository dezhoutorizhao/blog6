### 类的析构函数

1.   类的析构函数是类的特殊成员函数，会在每次删除所创建的对象时执行

2.   **析构函数的名称和类的名称是完全相同的**（类似构造器），只是**在前面加了个波浪号（~）作为前缀**，**不会返回任何值，也不能带有任何参数**

3.   ```c++
     Line();//这是构造函数的声明
     ~Line(); //这是析构函数的声明
     Line::Line(void){
         //这是构造器
     }
     Line::~Line(void){
         //这是析构函数
     }

## 友元函数

1.   类的友元函数是定义在**类的外部**，但**有权访问类的所有私有成员和保护成员**
2.   友元可以是一个函数，也可以是一个类，友元函数，友元类
3.   定义友元函数：类定义中该函数原型前使用关键字**friend**
4.   **友元函数可以访问类的私有成员，但是注意，友元函数不是类的成员函数**
5.   ![](https://strongwillpro.oss-cn-beijing.aliyuncs.com/img/20220926194231.png)

## 内联函数

1.   内联函数是直接复制镶嵌到主函数中去的，就是直接将内联函数的代码放到内联函数的位置上
2.   与一般函数的不同：主函数在调用一般函数时，是指令跳转到被调用函数的入口地址，执行完被调用函数后，指令再跳转回主函数上继续执行后面的代码
3.   ![](https://strongwillpro.oss-cn-beijing.aliyuncs.com/img/20220926203713.png)
4.   用inline定义内联函数，不允许使用循环，开关
5.   内联函数先定义后使用

## this指针

1.   每一个对象都可以通过this指针来访问自己的地址
2.   this指针是所有成员函数的隐含参数
3.   **友元函数没有this指针，因为友元不是类的成员，只有成员函数才有this指针**，记住，是**成员函数内部**，实参和形参的位置不能手动加
4.   友元函数，全局函数，static函数都不具备this指针

## 继承中构造函数的顺序

1.   子类中声明了构造函数（无参或带参），则子类的构造函数可以写成任何形式，不用顾及父类的构造函数
2.   构造函数先调用父类的，再调用子类的
3.   如果父类只声明了无参构造函数，没有显示的调用父类的构造，则会调用父类的无参构造函数，也就是说，父类的无参构造函数被隐式地调用，再调用子类的构造函数

## 析构函数的顺序

1.   #### 先调用父类的构造函数，再初始化成员，最后调用自己的构造函数

2.   #### 先调用自己的析构函数，再析构成员，最后调用父类的析构函数

3.   #### 如果父类定义了有参数的构造函数，则自己也必须自定义带参数的构造函数

## 静态数据成员可以不止通过 类名::标识符来调用

1.   ```c++
     #include<bits/stdc++.h>
     using namespace std;
     
     class Student {
         public:
             static int c; //声明一个变量为静态变量时，只需要在声明时使用static
     };
     
     int Student::c = 0; //不需要在初始化赋值（定义）的时候再次使用static关键字
     
     int main(void) {
         Student student;
         student.c = 1; // 在调用的时候，可以使用对象来调用，而不去使用类名进行调用
         cout << Student::c << endl;
         return 0;
     }

继承

  1. 基类构造函数的调用顺序和他们在派生类中的出现顺序（派生类中调用的基类构造函数）无关，而是和声明派生类时基类出现的顺序相同

  2. ```c++
    class D : public A , public B , public C{

      D(形参列表):A(实参列表),B(实参列表),C(){};

    }; //上面的构造器调用顺序及为D->A->B->C，和声明派生类时的基类出现的顺序相同，而和派生类的构造函数中谁先调用无关

    
3.   ### C++中，派生类的对象可以赋值给[基类](https://so.csdn.net/so/search?q=基类&spm=1001.2101.3001.7020)对象，因为，一个派生 [类对象](https://so.csdn.net/so/search?q=类对象&spm=1001.2101.3001.7020)，也可看作是一个基类对象（大学生是学生）。反过来则不行(学生未 必是大学生） 同样，调用需要基类对象作参数的函数时，以派生类对象作为实参，也是没有问题的:如果函数的参数是基类对象或基类对象的引用，函数调用时的实参可以是派生类对象。

4.   如果公有派生，则可以用派生类对象给基类对象赋值。

5.   派生类对象隐含转换为基类对象，即用派生类对象中从基类继承来的成员，逐个赋值给基类对象的成员。
     注意：public继承方式下，基类的private成员，在派生类中虽然无法访问，但是还是客观存在的！！！。如果一定要通过派生类对象访问基类的private成员，那么只能在基类中设置一个公有接口。
